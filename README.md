# 🎓 ThesEase: A Thesis Slot Booking System

## 📘 Project Overview

**Thes-ease** is a web-based Thesis Slot Booking System designed to simplify the process of browsing, scheduling and booking slots for students interested in doing thesis. The system addresses common challenges such as manual scheduling conflicts, lack of centralized coordination and inefficient information.

The platform enables students to book thesis slots of superviors, allows supervisors to manage their schedules and helps students to find the best suitable supervisor based on their interest. It will be a one stop solution for all the unnecessary confusion and hassle regarding the entire process. 

This project was developed as part of a **Software Engineering course**, following structured software development practices including requirement analysis, system modeling, documentation, and implementation planning.

---

## 🎯 Project Objectives

The primary objectives of Thes-ease are:

- To digitize and automate the thesis slot booking process
- To get information of thesis supervisors including their available slots   
- To minimize scheduling conflicts and overlapping bookings  
- To provide role-based access for students, supervisors, and administrators  
- To ensure transparency and accountability in thesis scheduling  
- To improve efficiency compared to traditional manual processes  

---

## 🧩 System Description

Thes-ease follows a structured system design with clearly defined user roles and responsibilities.

### 👨‍🎓 Student Module
- View available thesis presentation slots  
- Book preferred slots  
- View booking status  
- Receive confirmation notifications
- Able to rate and review supervisors  

### 👩‍🏫 Supervisor Module
- Monitor assigned students  
- Approve or manage bookings  
- Track schedules for consultations 

### 🛠️ Admin Module
- Approve requests from students and supervisors 
- Oversee all bookings  
- Prevent scheduling conflicts  
- Manage user roles and system settings  

The system is designed with scalability, usability, and maintainability in mind, sticking to core Software Engineering principles.
